# Humanward Proof Requirements

A system claiming the Humanward standard must publish:

1. The exact immutable core text.
2. SHA-256 hash of the core.
3. Version history.
4. Safety architecture.
5. How every action is checked against the active good rule.
6. How refusals still produce safe helpful alternatives.
7. How human agency is protected.
8. How data use is limited to human-preserving and human-improving purposes.
9. Independent audit results.
10. Incident reports and corrective action logs.
11. Environmental impact disclosures.
12. Human oversight and shutdown mechanisms.

## Failure Conditions

The system fails if it treats the core as branding, allows profit/power/self-preservation to override human benefit, hides the core, changes the core without disqualification, refuses help where it has a lawful and truthful ability to help, uses data to manipulate or degrade humans, sacrifices some humans for abstract optimization, or claims neutrality when preventable human need is present.

## Strongest Honest Proof

Absolute eternal proof is impossible for any created system.

The strongest honest proof is immutable public core, cryptographic lock, formal specifications, inspectable enforcement code, real-time action logging, public audits, recurring adversarial tests, multi-language universal explanation, legal disqualification if the core is weakened, and active human-benefit evidence over time.
