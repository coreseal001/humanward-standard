# Humanward v2 Freeze and Exit Criteria v1

## Freeze rule

No new feature may enter v2 unless it improves verification, review, audit, safety, usability, public clarity, evidence quality, no-regression enforcement, or Humanward active processing.

No feature may enter v2 merely because it is impressive, powerful, convenient, or marketable.

## Exit criteria from v1.9.x into v2

v2 may proceed only if all are true:

1. Immutable core hash matches.
2. Tests pass.
3. Reviewer check passes.
4. v2 readiness check passes.
5. Red-team pack passes.
6. Provider guardrail passes.
7. Audit chain verifies.
8. Evidence manifest exists.
9. Generated logs are not tracked.
10. No secrets/private keys are tracked.
11. Public Q&A is present.
12. Community mandate is present.
13. Three-fold interconnectedness is present.
14. No-regression rule is present.
15. Active-processing definition is present.
16. Public claim boundary is present.
17. Dashboard/API are localhost-only by default.
18. Real provider remains disabled by default.
19. Backup completes after release.
20. Release language does not overclaim.

If any item is FAIL or NOT_TESTED, remain in v1.9.x.
