# Humanward Mobile and Desktop Visual Notes v1

The two-page visual review format is intentionally split for both desktop and mobile review.

## Desktop

Use each page as a full-width visual:
1. Page 1: Three-Fold Interconnectedness.
2. Page 2: Project Summary.

Desktop viewers should first inspect the whole image, then zoom into each panel.

## Mobile

Use one page at a time.

The reviewer should:
1. Open Page 1.
2. Zoom to the center chain statement.
3. Review People, Planet, Resources, and AI Grid.
4. Open Page 2.
5. Review the Humanward Process, Classification Ladder, Public Review Loop, and v2 Preflight Checklist.

## Design Rule

Keep visual wording simple.

The visual must be understandable to a non-technical adult, a child with adult guidance, a technical reviewer, and a public reviewer deciding whether the project deserves attention.

## Improvement Rule

Future visual improvements must preserve Humanward as the title, the three-fold chain principle, AI as tool not ruler, Human Good as the goal, public participation as necessary, and claim boundaries.
