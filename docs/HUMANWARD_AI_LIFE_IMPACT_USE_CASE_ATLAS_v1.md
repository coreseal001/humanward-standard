# Humanward AI Life-Impact Use Case Atlas v1

## Purpose

This atlas records AI use cases that affect human life, safety, dignity, agency, livelihood, environment, and future wellbeing.

The goal is not fear. The goal is better classification, prevention, and Humanward improvement.

## Life-impact comparison table

| Domain | Positive use | Life-threatening or serious failure mode | Humanward prevention / alternative |
|---|---|---|---|
| Healthcare diagnosis | Earlier detection, triage support, image review, clinical decision support | Misdiagnosis, missed tumor, wrong dosage, biased triage, overreliance | Human review, uncertainty display, bias testing, audit, patient safety gate |
| Medical devices / surgery | Navigation support, monitoring, precision assistance | Misidentified anatomy, inaccurate guidance, adverse events, device recall | Device validation, fail-safe mode, surgeon override, incident reporting |
| Drug safety | Toxicity prediction, trial safety screening, early hazard discovery | False confidence, missed toxicity, unsafe trial decisions | Explainability, human toxicology review, conservative thresholds |
| Autonomous vehicles | Driver assistance, collision prevention, mobility access | Failure to detect pedestrians, unsafe test deployment, disabled safeguards | Safety case, real-world limits, emergency braking, independent investigation |
| Cybersecurity | Defensive detection, phishing analysis, malware triage | Malware generation, phishing automation, infrastructure attacks | Refuse misuse, defensive-only tooling, identity verification |
| Critical infrastructure | Predictive maintenance, grid/water/transport monitoring | Bad automation decisions, cyber-physical harm, cascading outage | Human-in-loop, fail-safe, restricted control, emergency rollback |
| Welfare / benefits / public services | Faster eligibility support, fraud detection, case triage | Wrong denial of essential services, bias, opaque appeals | Explanation, appeal rights, human review, fairness audit |
| Policing / security | Pattern review, workload support, missing-person search | Bias, wrongful targeting, illegal profiling, over-policing | Rights impact review, transparency, accountability, strict limits |
| Education | Tutoring, accessibility, translation, individual support | Manipulation, surveillance, unfair grading, dependency | Human teacher authority, privacy, explainable feedback |
| Employment | Matching, accessibility, skills analysis | Discrimination, hidden screening, livelihood harm | Audit, explainability, appeal, bias testing |
| Finance | Fraud detection, budgeting, risk monitoring | Wrong denial, predatory targeting, financial harm | Explainability, consent, appeal, suitability checks |
| Information / media | Translation, summarization, accessibility, public education | Misinformation, deepfakes, panic, manipulation | Provenance, labeling, verification, claim boundaries |
| Environment | Monitoring ecosystems, emissions, disasters, agriculture | Optimization that harms communities or ecology | Human/environment impact gate, public review, precaution |
| Military / weapons | Defensive logistics, demining, rescue support | Autonomous lethal targeting, escalation, mass harm | Human control, legal review, refusal of autonomous harm |

Any life-impacting AI must not be treated as ordinary software.
