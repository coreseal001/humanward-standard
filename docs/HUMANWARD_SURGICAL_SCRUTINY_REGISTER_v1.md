# Humanward Surgical Scrutiny Register v1

This register records known improvement areas before and after v2.

1. Pattern-based verifier limits: add semantic tests and adversarial mutation.
2. Audit integrity: add public-key signing and timestamping.
3. Provider connection: keep disabled until zero-trust boundary exists.
4. Dashboard/API security: keep localhost-only until authentication/security review exists.
5. Red-team breadth: expand from small pack to larger scenario library.
6. Formal verification: add property tests/invariants where practical.
7. Human review workflow: add triage status, reviewer notes, and disposition.
8. Evidence manifest: add signed manifests and release-linked checksums.
9. Public communications: add claim-firewall checks.
10. Environmental foundation: add environmental-impact classification where relevant.
11. Usefulness risk: backtest safe beneficial actions so Humanward does not become merely restrictive.
12. Community process: add issue templates, moderation rules, and safe disclosure channels.

Current acknowledgement: Humanward is strong enough for v1.9.x pre-v2 hardening. It is not yet enough to claim full compliance or production readiness. That restraint is part of the standard.
