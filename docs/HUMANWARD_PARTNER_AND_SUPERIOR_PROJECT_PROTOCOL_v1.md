# Humanward Partner and Superior Project Protocol v1

Date: 2026-06-09

## Purpose

This protocol defines what Humanward does when it finds similar or better work.

## If another project is better

If another project is stronger in a specific area, Humanward should:

1. Acknowledge it.
2. Study it.
3. Cite it where appropriate.
4. Ask whether collaboration is useful.
5. Adopt compatible improvements.
6. Avoid duplicating weaker work.
7. Preserve Humanward's immutable core.

## If another project conflicts with Humanward

If another project improves AI capability but weakens human agency, privacy, truth, lawfulness, or dignity, Humanward should not adopt that part.

Useful capability is not enough.

Capability must remain subordinate to Human Good.

## Partner-first posture

Humanward should treat aligned projects as potential allies, not enemies.

The best outcome is a larger ecosystem of AI systems and standards that preserve and empower humankind.

## No hostile competition rule

Humanward must not sabotage, harass, misrepresent, or attack other projects.

Critique must be evidence-based, lawful, and specific.

## Integration rule

A borrowed or inspired improvement must be:

- credited where appropriate,
- checked against the immutable core,
- tested,
- reviewed,
- documented,
- added only if it improves Humanward without regression.
